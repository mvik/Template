# Anti-Overengineering

## Purpose

Challenge unnecessary complexity, speculative scalability, and process growth.

## Trigger conditions

Run this skill when:

- new abstractions are proposed
- automation is added
- governance expands
- architecture becomes more layered
- a simple problem receives a large solution

## Checklist

- What concrete problem does this solve?
- Can the same outcome be achieved more simply?
- Is this solving a current problem or a hypothetical future one?
- Does complexity reduce enough future pain to justify itself?
- Can existing structure already support the need?

## Anti-patterns

- framework addiction
- speculative scalability
- architecture astronautics
- automation without ROI
- governance without measurable value
- platformization before product value exists

## Output

Emit:

- overengineering risk: low/medium/high
- unnecessary complexity identified
- smallest safe simplification

# Simplicity Preservation

## Purpose

Protect the project from unnecessary moving parts and irreversible complexity.

## Trigger conditions

Run this skill when:

- multiple new components are introduced
- workflows become harder to explain
- maintenance burden grows
- a simpler alternative may exist

## Checklist

- Can this be solved with fewer moving parts?
- Can existing mechanisms handle this already?
- Does this introduce long-term maintenance burden?
- Is this complexity reversible?
- Will this still feel understandable in six months?

## Anti-patterns

- unnecessary moving parts
- hidden operational burden
- irreversible architecture decisions
- complexity accumulation
- elegance through complication

## Output

Emit:

- simplicity status: preserved/at risk/degraded
- avoidable complexity identified
- recommended simplification

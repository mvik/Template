# Project Operating Model

How work is groomed and implemented in this project.

## Loop

1. **Identify** — surface the problem or opportunity as an epic or feature issue.
2. **Groom** — clarify intent, constraints, and acceptance criteria before any implementation begins.
3. **Build** — implement the smallest thing that satisfies the criteria.
4. **Validate** — confirm the outcome against acceptance criteria and the vision anchor.
5. **Log** — record key decisions in `DECISIONS.md`.

## Grooming checklist

Before starting implementation:
- [ ] Is the problem clearly stated?
- [ ] Does it align with `docs/vision.md`?
- [ ] Are acceptance criteria defined and verifiable?
- [ ] Is the scope explicitly bounded?
- [ ] Is there a simpler way to achieve the same outcome?

## Scope drift

If work expands beyond the original intent, stop and re-groom. Do not absorb scope silently.
